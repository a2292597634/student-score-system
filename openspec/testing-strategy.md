# 测试策略规范

## 一、测试架构

本项目为轻量 CLI 应用，测试分层简化为两层：

- **单元测试**：覆盖 `src/*.js` 每个模块的导出函数（纯逻辑、数据校验、统计计算）
- **集成测试**：覆盖模块间协作（storage → student 数据流、CLI 命令到数据操作）

## 二、测试工具

- Vitest（无需 jsdom，CLI 应用无浏览器依赖）

## 三、TDD 执行流程

每个功能点必须按以下三步执行：

### Step 1: Red（写失败测试）
- 先写测试用例，描述期望行为
- 运行测试，确认测试失败

### Step 2: Green（写最小实现）
- 写最小代码使测试通过
- 禁止在使测试通过之前写"额外"的代码

### Step 3: Refactor（重构）
- 在测试保护下重构代码
- 每次重构后运行测试，确保没有破坏已有功能

## 四、测试目录规范

```
tests/
├── storage.test.js       # 存储引擎测试
├── student.test.js       # 学生管理测试
└── integration/          # 集成测试
    └── cli-flow.test.js  # CLI 完整流程测试
```

## 五、覆盖率检查

每个模块必须有对应测试文件，每个测试文件至少 3 个测试用例，测试通过率 100%。
