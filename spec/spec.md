# 核心规范文档

## 一、项目概述

简易学生成绩管理系统（CLI 应用），面向教务人员日常使用，实现学生成绩全流程管理能力。

## 二、整体业务需求

### 2.1 项目架构与持久化能力
- 完成项目规范化初始化，搭建完整目录结构
- 实现 JSON 文件数据库持久化能力
- 实现学生数据读写、数据初始化能力

### 2.2 学生成绩录入与数据校验
- 支持录入学生核心信息：唯一学号、姓名、语文/数学/英语三科成绩
- 完善数据合法性校验：学号非空唯一、成绩为0-100有效数字
- 重复学号自动拦截，避免数据重复录入

### 2.3 学生成绩查询能力
- 支持通过学号精准查询单个学生完整信息及各科成绩
- 支持查询全部已录入学生的成绩列表，批量展示数据
- 异常场景友好提示：学号不存在、无任何录入数据时给出清晰提示

### 2.4 成绩统计与全局容错优化
- 核心统计能力：单学生总分、平均分计算展示，班级单科平均分统计
- 全局容错处理：非法文字输入、空数据查询、异常参数拦截
- 所有操作数据自动持久化，程序重启后数据不丢失

## 三、功能规格（按 Change 拆分）

### Change 1: 项目初始化与数据持久化

#### Requirement: JSON 文件数据读取
系统 SHALL 从 `data/students.json` 读取学生数据，文件不存在时自动初始化。

**输入规范：** 无（函数无参数）

**输出规范：** 返回解析后的学生数组，文件不存在时返回空数组 `[]`

**业务规则：**
- 数据目录和文件自动创建
- JSON 解析失败时返回空数组并提示

**异常场景：**
- 文件不存在 → 自动初始化
- JSON 格式损坏 → 返回空数组，提示用户

**验收标准：** `loadStudents()` 可正确读取已保存数据，文件不存在时不报错

#### Requirement: JSON 文件数据写入
系统 SHALL 将学生数据写入 `data/students.json`，使用格式化 JSON。

**输入规范：** `Array` 类型的学生数据数组

**输出规范：** 无返回值，文件写入成功或抛出异常

**业务规则：**
- 使用 2 空格缩进格式化 JSON
- 覆盖式写入（非追加）

**异常场景：**
- 写入权限不足 → 抛出异常并提示

**验收标准：** `saveStudents(data)` 写入后，`loadStudents()` 可读取到相同数据

### Change 2: 学生成绩录入与数据校验

#### Requirement: 学生信息录入
系统 SHALL 支持录入学号、姓名、三科成绩，校验通过后持久化保存。

**输入规范：**
```
{ id: "非空字符串", name: "非空字符串", chinese: 0-100, math: 0-100, english: 0-100 }
```

**输出规范：**
```
{ success: true|false, message: "中文提示信息", data: 学生对象|null }
```

**业务规则：**
- 学号非空且唯一
- 姓名非空
- 三科成绩为 0-100 的整数
- 重复学号自动拦截

**异常场景：**
- 学号为空 → 提示"学号不能为空"
- 学号已存在 → 提示"该学号已存在，不允许重复录入"
- 姓名为空 → 提示"姓名不能为空"
- 成绩超出范围 → 提示"成绩必须在0-100之间"
- 成绩非数字 → 提示"成绩必须为有效数字"

**验收标准：** 合法数据录入成功，非法数据被拦截并给出中文提示

### Change 3: 学生成绩查询能力

#### Requirement: 学号精准查询
系统 SHALL 支持通过学号查询单个学生的完整信息。

**输入规范：** 学号字符串

**输出规范：**
```
{ success: true|false, message: "...", data: 学生对象|null }
```

**业务规则：**
- 精确匹配学号
- 未找到时给出友好提示
- 展示学生全部信息（学号、姓名、三科成绩）

**异常场景：**
- 学号不存在 → 提示"未找到学号为 xxx 的学生"
- 无任何录入数据 → 提示"暂无学生数据，请先录入"

**验收标准：** 存在学号返回完整信息，不存在的学号给出清晰提示

#### Requirement: 全部学生列表查询
系统 SHALL 支持查询全部已录入学生的成绩列表。

**输入规范：** 无

**输出规范：**
```
{ success: true, message: "...", data: 学生数组 }
```

**业务规则：**
- 以列表形式展示学号、姓名、三科成绩
- 无数据时给出提示

**异常场景：**
- 无任何录入数据 → 提示"暂无学生数据，请先录入"

**验收标准：** 返回完整学生列表，无数据时给出友好提示

### Change 4: 成绩统计与全局容错优化

#### Requirement: 单学生成绩统计
系统 SHALL 支持计算单个学生总分和平均分。

**输入规范：** 学生对象

**输出规范：**
```
{ total: Number, average: Number(保留1位小数) }
```

**业务规则：**
- 总分 = 语文 + 数学 + 英语
- 平均分 = 总分 / 3，保留1位小数

**验收标准：** 计算结果正确

#### Requirement: 班级单科平均分
系统 SHALL 支持计算班级各科平均分。

**输入规范：** 学生数组

**输出规范：**
```
{ chinese: Number, math: Number, english: Number }
```

**业务规则：**
- 单科平均分 = 该科总分 / 学生人数
- 保留1位小数
- 无学生时各项返回 0

**异常场景：**
- 无学生数据 → 各项返回 0，提示"暂无学生数据"

**验收标准：** 计算结果正确，无数据时返回 0

#### Requirement: 全局容错处理
系统 SHALL 对所有交互入口进行输入校验和异常保护。

**业务规则：**
- 菜单选择非数字或超出范围时给出提示
- 非法文字输入替换数字输入时给出提示
- 所有异常不导致程序崩溃，回到主菜单

**验收标准：** 各种非法输入和边界场景不会导致程序崩溃

#### Requirement: 自动持久化
系统 SHALL 在每次数据变更操作后自动将数据写入文件。

**业务规则：**
- 录入成功后自动写入
- 程序重启后数据不丢失

**验收标准：** 录入数据后重启程序，数据依然存在

### Change 5: 学生信息修改与删除

#### Requirement: 修改学生信息
系统 SHALL 支持通过学号修改已有学生的姓名和成绩，未提供的字段保持原值不变。

**输入规范：**
```
students: Array, id: String, fields: {name?, chinese?, math?, english?}
```

**输出规范：**
```
{ success: true|false, message: "中文提示信息", data: 更新后的学生对象|null }
```

**业务规则：**
- 部分更新，仅修改提供的字段
- 修改后仍需通过完整数据校验
- 学号不可修改（唯一标识）

**异常场景：**
- 学号不存在 → 提示"未找到学号为 xxx 的学生"
- 修改后数据不合法 → 提示具体校验错误，原数据不变
- 无任何数据 → 提示"暂无学生数据，请先录入"

**验收标准：** 可修改姓名和任意科目成绩，非法修改被拦截且原数据不变

#### Requirement: 删除学生记录
系统 SHALL 支持通过学号删除学生记录，删除后数据自动持久化。

**输入规范：** 学号字符串

**输出规范：**
```
{ success: true|false, message: "...", data: 被删学生对象|null }
```

**业务规则：**
- 删除前需用户确认
- 删除后自动持久化

**异常场景：**
- 学号不存在 → 提示"未找到学号为 xxx 的学生"
- 无任何数据 → 提示"暂无学生数据，请先录入"

**验收标准：** 确认后成功删除，学号不存在时给出友好提示
